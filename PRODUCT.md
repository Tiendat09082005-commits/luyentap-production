# Product

## Register

product

## Users

Admin staff of a Vietnamese e-commerce store selling technology products (phones, laptops, accessories). They manage inventory, process orders, handle customer support chat, and configure site settings. Typically using a desktop browser (1280px+) during business hours in a well-lit office environment. They are not developers; they need clear, functional interfaces that get out of their way.

## Product Purpose

TidaAdmin is the back-office admin panel for an e-commerce platform. It provides CRUD management for products, brands, product categories, attributes, orders, user accounts, roles/permissions, real-time chat support, and site-wide settings. Success means: admin staff can complete any management task quickly, with confidence, without training.

## Brand Personality

Competent, calm, efficient. The interface should feel like a well-organized workspace: everything in its place, no visual noise, no ambiguity about what to do next.

## Anti-references

- Overly decorative SaaS dashboards with gradient cards, animated counters, and glassmorphism everywhere.
- Dark-themed admin panels that prioritize "cool" aesthetics over readability during daytime use.
- Bootstrap default styling with no personality (generic grey cards, default blue buttons).
- Cluttered admin panels where every metric fights for attention equally.

## Design Principles

1. **Clarity over decoration**: Every visual element must communicate function. If removing it changes nothing, remove it.
2. **Consistent rhythm**: Spacing, typography, and component shapes should be predictable across all pages so staff build muscle memory.
3. **Quiet confidence**: The interface should feel professional without trying hard. No effects that say "look at me."
4. **Progressive disclosure**: Show the primary action and essential data first; details on demand.
5. **Forgiveness**: Destructive actions (delete, status change) always confirm. Errors are clear and recoverable.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum contrast ratios.
- All interactive elements keyboard-accessible.
- Vietnamese language throughout; UI copy should be concise and unambiguous.
- No reliance on color alone to convey status (pair with icons or text labels).
